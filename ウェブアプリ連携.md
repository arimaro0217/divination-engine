デプロイの手順を詳細に案内します。

以下の手順で進めてください。

1. GitHubにpush
まず、ローカルでGitのコミットを行います。私はGitの初期化とファイルのステージング（git add .）まで行いました。 あなたの環境でGitのユーザー設定がまだの場合、コミット時にエラーになるため、以下のコマンドを実行してください。

(ターミナルで実行)

powershell
# 1. ユーザー設定（もし未設定なら）
git config --global user.name "あなたの名前"
git config --global user.email "your@email.com"
# 2. コミット
git commit -m "Initial commit"
# 3. GitHubで新しいリポジトリを作成してください（空のリポジトリ）
# 4. リモートを追加してpush（URLは作成したリポジトリのものに変えてください）
git remote add origin https://github.com/あなたのユーザー名/リポジトリ名.git
git branch -M main
git push -u origin main


2. Renderでバックエンドデプロイ
Render Dashboard にアクセスし、New Web Service を作成します。
GitHubリポジトリを接続します。
以下の設定を入力します：
Name: 任意の名前 (例: divination-api)
Root Directory: divination_engine
Runtime: Python 3
Build Command: pip install -r requirements.txt
Start Command: uvicorn api_server:app --host 0.0.0.0 --port $PORT
Create Web Service をクリック。
デプロイ完了後、発行されたURL (例: https://divination-api.onrender.com) をコピーしておきます。


3. Vercelでフロントエンドデプロイ (初回)
ターミナルで webapp ディレクトリに移動します。
powershell
cd webapp
Vercelにデプロイします（Vercel CLIがインストールされている前提ですが、npxでも可能です）。
powershell
npx vercel
セットアップの質問には基本的に Enter (Y) で答えていきます。
Project Nameなどは任意で設定。
Want to modify these settings? は No でOK（静的サイトとして認識されます）。


4. api-client.jsの本番URLを設定
RenderのURLが確定したら、フロントエンドの接続先を更新します。

ファイル webapp/js/api-client.js を開きます。
10行目付近の本番用URLを書き換えます。
javascript
/* webapp/js/api-client.js */
    BASE_URL: window.location.hostname === 'localhost'
        ? 'http://localhost:8000/api'
        : 'https://あなたのRenderアプリ名.onrender.com/api', // ←ここを書き換え
変更を適用するために、再度Vercelにデプロイします（本番用）。
powershell
npx vercel --prod
5. 動作確認
Vercelで発行されたURLにアクセスし、占いの計算が正しく行われるか確認してください。

この手順で進めてください。何かエラーが出たら教えてください。 まずは Step 1: GitHubへのPush からお願いします。コミットとPushができたら教えてください。
